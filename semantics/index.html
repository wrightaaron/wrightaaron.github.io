Semantic Tags

What's in a name?

Why?

When you want to make sure the your intentions are understood, it is better to state them explicitly instead of implicitly. If you aren't clear with your intentions, it leaves what you say and do open to interpretation and can lead to unexpected implications. While it is better to ask what is meant by something when we are unsure, that question doesn't always get asked. Sometime, it can't be asked because the person that knows the answer is not available. When we write our code, we should do so with the expectation that someone else may be working on it at some point in the future and different types of software will be rendering that code. That's where using semantic HTML tags comes in. Using meaningful tag names can make a huge difference in the way our code is understood. They will help us be more explicit with our intentions and leave less up to interpretation.

Definition

:of or relating to meaning in language
Merriam-Webster

Example

We use semantic elements when writing our HTML because of the meaning it gives those elements. When adding a header to a block of content in our html, using 

<section> 
	<article> 
		<header> 
			<h1> 
				This is my heading 
			</h1> 
		</header> 
	</ article> 
</ section>

gives us more meaning than using a div with a class like 

<div class="section"> 
	<div class="article"> 
		<div class="header"> 
			<h1> 
				This is my heading 
			</h1> 
		</div> 
	</ div> 
</ div>

This allows the browser, screen readers, and other developers more easily understand what the intended structure of our page is.

HTML 5 introduced several new semantic elements like header, footer, article, and many others.
