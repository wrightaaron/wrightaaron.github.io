Using fonts in our web pages

Web-safe Fonts

There are a number of fonts available that are considered web-safe. This is because they are standard fonts that most browsers should have available to them. However, just because a font should be web-safe, doesn't mean you should only declare one and leave it at that. If you declare one font family and no other options, visitors to your page may not be getting the experience you want them to have. That is why we have the ability to declare multiple similaer font families in our CSS. If the first font is not available, maybe the next one will be. If all else fails, we can tell the browser to use a generic serif or sans-serif font.

Definition

A font is a collection of characters with a similar design.

Serif vs Sans-serif

You may or may not have heard the terms serif and sans-serif before. A serif is a small extension at the edge of a character. You may have seen this on letters like T or M. "Sans" means without so sans-serif means without those decorative extensions on the characters like T or M. You surely noticed that I used the same characters in both of those examples. They are however using different fonts do demonstrate how those characters look with and without the serifs.

In the CSS

Below you can see some examples of setting a font-family in your CSS file. In the examples below, we are using three options for each font-family value. The browser will try the first one. If the first one is available, that's great. If it is not available, the browser will then try the next one. This goes on until the browser finds one it can render or goes to the default font for the browser. That is why it is good to end our font-family declarations with a generic font style to be used such as serif or sans-serif like the examples below. This tells the browser that if it can't find any of the font's listed, use a generic one that is the style we gave it. That way you are at least getting the serif, sans-serif, cursive, etc. that you wanted. 

font-family:"Times New Roman", Times, serif; 
font-family:Arial, Helvetica, sans-serif;

We can use as many fonts as needed in the value for the font-family attribute. They just need to be separated by commas. Another thing I would like to point out is the use of the quotation marks for "Times New Roman" Because the name of that font is multiple words with spaces, it must be inside quotation marks. As long as the font is one word, it does not need the quotes but they are required when using a multi-word font name.
